## ಪೈಪ್ಲೈನ್ಗಳೊಂದಿಗೆ ಕೆಲಸ ಮಾಡುವುದು[[ಪೈಪ್ಲೈನ್ಗಳೊಂದಿಗೆ ಕೆಲಸ ಮಾಡುವುದು]]

<Youtube id="tiZFewofSLM" />

🤗 ಟ್ರಾನ್ಸ್‌ಫಾರ್ಮರ್ಸ್ ಲೈಬ್ರರಿಯಲ್ಲಿರುವ ಅತ್ಯಂತ ಮೂಲಭೂತ ವಸ್ತು `ಪೈಪ್‌ಲೈನ್()` ಕಾರ್ಯವಾಗಿದೆ. ಇದು ಮಾದರಿಯನ್ನು ಅದರ ಅಗತ್ಯ ಪೂರ್ವ-ಸಂಸ್ಕರಣೆ ಮತ್ತು ನಂತರದ-ಸಂಸ್ಕರಣಾ ಹಂತಗಳೊಂದಿಗೆ ಸಂಪರ್ಕಿಸುತ್ತದೆ, ಇದು ಯಾವುದೇ ಪಠ್ಯವನ್ನು ನೇರವಾಗಿ ಇನ್‌ಪುಟ್ ಮಾಡಲು ಮತ್ತು ಅರ್ಥವಾಗುವ ಉತ್ತರವನ್ನು ಪಡೆಯಲು ನಮಗೆ ಅನುವು ಮಾಡಿಕೊಡುತ್ತದೆ:

```python
from transformers import pipeline

classifier = pipeline("sentiment-analysis")
classifier("I've been waiting for a HuggingFace course my whole life.")
```

```python out
[{'label': 'POSITIVE', 'score': 0.9598047137260437}]
```

ನಾವು ಹಲವಾರು ವಾಕ್ಯಗಳನ್ನು ಸಹ ರವಾನಿಸಬಹುದು!

```python
classifier(
    ["I've been waiting for a HuggingFace course my whole life.", "I hate this so much!"]
)
```

```python out
[{'label': 'POSITIVE', 'score': 0.9598047137260437},
 {'label': 'NEGATIVE', 'score': 0.9994558095932007}]
```

ಪೂರ್ವನಿಯೋಜಿತವಾಗಿ, ಈ ಪೈಪ್‌ಲೈನ್ ಇಂಗ್ಲಿಷ್‌ನಲ್ಲಿ ಭಾವನೆ ವಿಶ್ಲೇಷಣೆಗಾಗಿ ಉತ್ತಮವಾಗಿ ಟ್ಯೂನ್ ಮಾಡಲಾದ ನಿರ್ದಿಷ್ಟ ಪೂರ್ವ-ತರಬೇತಿ ಪಡೆದ ಮಾದರಿಯನ್ನು ಆಯ್ಕೆ ಮಾಡುತ್ತದೆ. ನೀವು `classifier` ವಸ್ತುವನ್ನು ರಚಿಸಿದಾಗ ಮಾದರಿಯನ್ನು ಡೌನ್‌ಲೋಡ್ ಮಾಡಲಾಗುತ್ತದೆ ಮತ್ತು ಸಂಗ್ರಹಿಸಲಾಗುತ್ತದೆ. ನೀವು ಆಜ್ಞೆಯನ್ನು ಮರು-ರನ್ ಮಾಡಿದರೆ, ಬದಲಿಗೆ ಸಂಗ್ರಹಗೊಂಡ ಮಾದರಿಯನ್ನು ಬಳಸಲಾಗುತ್ತದೆ ಮತ್ತು ಮಾದರಿಯನ್ನು ಮತ್ತೆ ಡೌನ್‌ಲೋಡ್ ಮಾಡುವ ಅಗತ್ಯವಿಲ್ಲ.

ನೀವು ಕೆಲವು ಪಠ್ಯವನ್ನು ಪೈಪ್‌ಲೈನ್‌ಗೆ ರವಾನಿಸಿದಾಗ ಮೂರು ಮುಖ್ಯ ಹಂತಗಳಿವೆ:

1. ಪಠ್ಯವನ್ನು ಮಾದರಿಯು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದಾದ ಸ್ವರೂಪಕ್ಕೆ ಪೂರ್ವ-ಸಂಸ್ಕರಿಸಲಾಗುತ್ತದೆ.
2. ಪೂರ್ವ-ಸಂಸ್ಕರಿಸಿದ ಇನ್‌ಪುಟ್‌ಗಳನ್ನು ಮಾದರಿಗೆ ರವಾನಿಸಲಾಗುತ್ತದೆ.
3. ಮಾದರಿಯ ಭವಿಷ್ಯವಾಣಿಗಳು ನಂತರದ-ಸಂಸ್ಕರಣೆಯಾಗಿರುತ್ತವೆ, ಆದ್ದರಿಂದ ನೀವು ಅವುಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು.

## ವಿಭಿನ್ನ ವಿಧಾನಗಳಿಗೆ ಲಭ್ಯವಿರುವ ಪೈಪ್‌ಲೈನ್‌ಗಳು

`ಪ್‌ಲೈನ್()` ಕಾರ್ಯವು ಬಹು ವಿಧಾನಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ, ಇದು ಪಠ್ಯ, ಚಿತ್ರಗಳು, ಆಡಿಯೋ ಮತ್ತು ಮಲ್ಟಿಮೋಡಲ್ ಕಾರ್ಯಗಳೊಂದಿಗೆ ಕೆಲಸ ಮಾಡಲು ನಿಮಗೆ ಅನುವು ಮಾಡಿಕೊಡುತ್ತದೆ. ಈ ಕೋರ್ಸ್‌ನಲ್ಲಿ ನಾವು ಪಠ್ಯ ಕಾರ್ಯಗಳ ಮೇಲೆ ಕೇಂದ್ರೀಕರಿಸುತ್ತೇವೆ, ಆದರೆ ಟ್ರಾನ್ಸ್‌ಫಾರ್ಮರ್ ಆರ್ಕಿಟೆಕ್ಚರ್‌ನ ಸಾಮರ್ಥ್ಯವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಉಪಯುಕ್ತವಾಗಿದೆ, ಆದ್ದರಿಂದ ನಾವು ಅದನ್ನು ಸಂಕ್ಷಿಪ್ತವಾಗಿ ವಿವರಿಸುತ್ತೇವೆ.

ಲಭ್ಯವಿರುವ ವಿಷಯಗಳ ಅವಲೋಕನ ಇಲ್ಲಿದೆ:

> [!TIP]
> ಪೈಪ್‌ಲೈನ್‌ಗಳ ಪೂರ್ಣ ಮತ್ತು ನವೀಕರಿಸಿದ ಪಟ್ಟಿಗಾಗಿ, [🤗 ಟ್ರಾನ್ಸ್‌ಫಾರ್ಮರ್‌ಗಳ ದಸ್ತಾವೇಜನ್ನು](https://huggingface.co/docs/hub/en/models-tasks) ನೋಡಿ.

### ಪಠ್ಯ ಪೈಪ್ಲೈನ್ಗಳು

- `text-generation`: ಪ್ರಾಂಪ್ಟ್‌ನಿಂದ ಪಠ್ಯವನ್ನು ರಚಿಸಿ
- `text-classification`: ಪಠ್ಯವನ್ನು ಪೂರ್ವನಿರ್ಧರಿತ ವರ್ಗಗಳಾಗಿ ವರ್ಗೀಕರಿಸಿ
- `summarization`: ಪ್ರಮುಖ ಮಾಹಿತಿಯನ್ನು ಸಂರಕ್ಷಿಸುತ್ತಾ ಪಠ್ಯದ ಚಿಕ್ಕ ಆವೃತ್ತಿಯನ್ನು ರಚಿಸಿ.
- `translation`: ಒಂದು ಭಾಷೆಯಿಂದ ಇನ್ನೊಂದು ಭಾಷೆಗೆ ಪಠ್ಯವನ್ನು ಅನುವಾದಿಸಿ
- `zero-shot-classification`: ನಿರ್ದಿಷ್ಟ ಲೇಬಲ್‌ಗಳ ಕುರಿತು ಪೂರ್ವ ತರಬೇತಿಯಿಲ್ಲದೆ ಪಠ್ಯವನ್ನು ವರ್ಗೀಕರಿಸಿ.
- `feature-extraction`: ಪಠ್ಯದ ವೆಕ್ಟರ್ ಪ್ರಾತಿನಿಧ್ಯಗಳನ್ನು ಹೊರತೆಗೆಯಿರಿ

### ಚಿತ್ರ ಪೈಪ್ಲೈನ್ಗಳು

- `image-to-text`: ಚಿತ್ರಗಳ ಪಠ್ಯ ವಿವರಣೆಯನ್ನು ರಚಿಸಿ
- `image-classification`: ಚಿತ್ರದಲ್ಲಿರುವ ವಸ್ತುಗಳನ್ನು ಗುರುತಿಸಿ
- `object-detection`: ಚಿತ್ರಗಳಲ್ಲಿ ವಸ್ತುಗಳನ್ನು ಪತ್ತೆ ಮಾಡಿ ಮತ್ತು ಗುರುತಿಸಿ

### ಆಡಿಯೋ ಪೈಪ್ಲೈನ್ಗಳು

- `automatic-speech-recognition`:ಭಾಷಣವನ್ನು ಪಠ್ಯಕ್ಕೆ ಪರಿವರ್ತಿಸಿ
- `audio-classification`: ಆಡಿಯೊವನ್ನು ವರ್ಗಗಳಾಗಿ ವರ್ಗೀಕರಿಸಿ
- `text-to-speech`: ಪಠ್ಯವನ್ನು ಮಾತನಾಡುವ ಆಡಿಯೊಗೆ ಪರಿವರ್ತಿಸಿ

### ಬಹುಮಾದರಿ ಪೈಪ್ಲೈನ್ಗಳು

- `image-text-to-text`: ಪಠ್ಯ ಪ್ರಾಂಪ್ಟ್ ಆಧರಿಸಿ ಚಿತ್ರಕ್ಕೆ ಪ್ರತಿಕ್ರಿಯಿಸಿ

ಈ ಪೈಪ್‌ಲೈನ್‌ಗಳಲ್ಲಿ ಕೆಲವನ್ನು ಹೆಚ್ಚು ವಿವರವಾಗಿ ಅನ್ವೇಷಿಸೋಣ!

## ಝೀರೋ-ಶಾಟ್ ವರ್ಗೀಕರಣ[[ಝೀರೋ-ಶಾಟ್ ವರ್ಗೀಕರಣ]]

ಲೇಬಲ್ ಮಾಡದ ಪಠ್ಯಗಳನ್ನು ವರ್ಗೀಕರಿಸಬೇಕಾದ ಹೆಚ್ಚು ಸವಾಲಿನ ಕೆಲಸವನ್ನು ನಿಭಾಯಿಸುವ ಮೂಲಕ ನಾವು ಪ್ರಾರಂಭಿಸುತ್ತೇವೆ. ಇದು ನೈಜ-ಪ್ರಪಂಚದ ಯೋಜನೆಗಳಲ್ಲಿ ಸಾಮಾನ್ಯ ಸನ್ನಿವೇಶವಾಗಿದೆ ಏಕೆಂದರೆ ಪಠ್ಯವನ್ನು ಟಿಪ್ಪಣಿ ಮಾಡುವುದು ಸಾಮಾನ್ಯವಾಗಿ ಸಮಯ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ ಮತ್ತು ಡೊಮೇನ್ ಪರಿಣತಿಯ ಅಗತ್ಯವಿರುತ್ತದೆ. ಈ ಬಳಕೆಯ ಸಂದರ್ಭದಲ್ಲಿ, `ಶೂನ್ಯ-ಶಾಟ್-ವರ್ಗೀಕರಣ `ಪೈಪ್‌ಲೈನ್ ತುಂಬಾ ಶಕ್ತಿಶಾಲಿಯಾಗಿದೆ: ವರ್ಗೀಕರಣಕ್ಕಾಗಿ ಯಾವ ಲೇಬಲ್‌ಗಳನ್ನು ಬಳಸಬೇಕೆಂದು ನಿರ್ದಿಷ್ಟಪಡಿಸಲು ಇದು ನಿಮಗೆ ಅನುಮತಿಸುತ್ತದೆ, ಆದ್ದರಿಂದ ನೀವು ಪೂರ್ವ-ತರಬೇತಿ ಪಡೆದ ಮಾದರಿಯ ಲೇಬಲ್‌ಗಳನ್ನು ಅವಲಂಬಿಸಬೇಕಾಗಿಲ್ಲ. ಆ ಎರಡು ಲೇಬಲ್‌ಗಳನ್ನು ಬಳಸಿಕೊಂಡು ಮಾದರಿಯು ವಾಕ್ಯವನ್ನು ಧನಾತ್ಮಕ ಅಥವಾ ಋಣಾತ್ಮಕ ಎಂದು ಹೇಗೆ ವರ್ಗೀಕರಿಸಬಹುದು ಎಂಬುದನ್ನು ನೀವು ಈಗಾಗಲೇ ನೋಡಿದ್ದೀರಿ - ಆದರೆ ನೀವು ಇಷ್ಟಪಡುವ ಯಾವುದೇ ಇತರ ಲೇಬಲ್‌ಗಳನ್ನು ಬಳಸಿಕೊಂಡು ಪಠ್ಯವನ್ನು ವರ್ಗೀಕರಿಸಬಹುದು.

```python
from transformers import pipeline

classifier = pipeline("zero-shot-classification")
classifier(
    "This is a course about the Transformers library",
    candidate_labels=["education", "politics", "business"],
)
```

```python out
{'sequence': 'This is a course about the Transformers library',
 'labels': ['education', 'business', 'politics'],
 'scores': [0.8445963859558105, 0.111976258456707, 0.043427448719739914]}
```

ಈ ಪೈಪ್‌ಲೈನ್ ಅನ್ನು _ಝೀರೋ-ಶಾಟ್_ ಎಂದು ಕರೆಯಲಾಗುತ್ತದೆ ಏಕೆಂದರೆ ಅದನ್ನು ಬಳಸಲು ನಿಮ್ಮ ಡೇಟಾದಲ್ಲಿನ ಮಾದರಿಯನ್ನು ನೀವು ಫೈನ್-ಟ್ಯೂನ್ ಮಾಡುವ ಅಗತ್ಯವಿಲ್ಲ. ಇದು ನಿಮಗೆ ಬೇಕಾದ ಯಾವುದೇ ಲೇಬಲ್‌ಗಳ ಪಟ್ಟಿಗೆ ಸಂಭವನೀಯತೆಯ ಸ್ಕೋರ್‌ಗಳನ್ನು ನೇರವಾಗಿ ಹಿಂತಿರುಗಿಸಬಹುದು!

> [!TIP]
> ✏️ **ಇದನ್ನು ಪ್ರಯತ್ನಿಸಿ!** ನಿಮ್ಮ ಸ್ವಂತ ಅನುಕ್ರಮಗಳು ಮತ್ತು ಲೇಬಲ್‌ಗಳೊಂದಿಗೆ ಆಟವಾಡಿ ಮತ್ತು ಮಾದರಿ ಹೇಗೆ ವರ್ತಿಸುತ್ತದೆ ಎಂಬುದನ್ನು ನೋಡಿ.


## ಪಠ್ಯ ಉತ್ಪಾದನೆ[[ಪಠ್ಯ ಉತ್ಪಾದನೆ]]

ಈಗ ಪೈಪ್‌ಲೈನ್ ಬಳಸಿ ಪಠ್ಯವನ್ನು ಹೇಗೆ ರಚಿಸುವುದು ಎಂದು ನೋಡೋಣ. ಇಲ್ಲಿ ಮುಖ್ಯ ಉದ್ದೇಶವೆಂದರೆ ನೀವು ಪ್ರಾಂಪ್ಟ್ ಅನ್ನು ಒದಗಿಸುತ್ತೀರಿ ಮತ್ತು ಮಾದರಿಯು ಉಳಿದ ಪಠ್ಯವನ್ನು ರಚಿಸುವ ಮೂಲಕ ಅದನ್ನು ಸ್ವಯಂ-ಪೂರ್ಣಗೊಳಿಸುತ್ತದೆ. ಇದು ಅನೇಕ ಫೋನ್‌ಗಳಲ್ಲಿ ಕಂಡುಬರುವ ಭವಿಷ್ಯಸೂಚಕ ಪಠ್ಯ ವೈಶಿಷ್ಟ್ಯವನ್ನು ಹೋಲುತ್ತದೆ. ಪಠ್ಯ ರಚನೆಯು ಯಾದೃಚ್ಛಿಕತೆಯನ್ನು ಒಳಗೊಂಡಿರುತ್ತದೆ, ಆದ್ದರಿಂದ ಕೆಳಗೆ ತೋರಿಸಿರುವಂತೆ ನೀವು ಅದೇ ಫಲಿತಾಂಶಗಳನ್ನು ಪಡೆಯದಿದ್ದರೆ ಅದು ಸಾಮಾನ್ಯವಾಗಿದೆ.

```python
from transformers import pipeline

generator = pipeline("text-generation")
generator("In this course, we will teach you how to")
```

```python out
[{'generated_text': 'In this course, we will teach you how to understand and use '
                    'data flow and data interchange when handling user data. We '
                    'will be working with one or more of the most commonly used '
                    'data flows — data flows of various types, as seen by the '
                    'HTTP'}]
```

`num_return_sequences` ಎಂಬ ವಾದದೊಂದಿಗೆ ಎಷ್ಟು ವಿಭಿನ್ನ ಅನುಕ್ರಮಗಳು ಉತ್ಪತ್ತಿಯಾಗುತ್ತವೆ ಎಂಬುದನ್ನು ಮತ್ತು `max_length` ಎಂಬ ವಾದದೊಂದಿಗೆ ಔಟ್‌ಪುಟ್ ಪಠ್ಯದ ಒಟ್ಟು ಉದ್ದವನ್ನು ನೀವು ನಿಯಂತ್ರಿಸಬಹುದು.

> [!TIP]
> ✏️ **ಇದನ್ನು ಪ್ರಯತ್ನಿಸಿ!** `num_return_sequences` ಮತ್ತು `max_length` ಎಂಬ ವಾದಗಳನ್ನು ಬಳಸಿ ತಲಾ 15 ಪದಗಳ ಎರಡು ವಾಕ್ಯಗಳನ್ನು ರಚಿಸಿ.

## ಪೈಪ್‌ಲೈನ್‌ನಲ್ಲಿ ಹಬ್‌ನಿಂದ ಯಾವುದೇ ಮಾದರಿಯನ್ನು ಬಳಸುವುದು[[ಪೈಪ್‌ಲೈನ್‌ನಲ್ಲಿ ಹಬ್‌ನಿಂದ ಯಾವುದೇ ಮಾದರಿಯನ್ನು ಬಳಸುವುದು]]

ಹಿಂದಿನ ಉದಾಹರಣೆಗಳು ಕೈಯಲ್ಲಿರುವ ಕಾರ್ಯಕ್ಕೆ ಡೀಫಾಲ್ಟ್ ಮಾದರಿಯನ್ನು ಬಳಸಿದ್ದವು, ಆದರೆ ನೀವು ಹಬ್‌ನಿಂದ ನಿರ್ದಿಷ್ಟ ಮಾದರಿಯನ್ನು ನಿರ್ದಿಷ್ಟ ಕಾರ್ಯಕ್ಕಾಗಿ ಪೈಪ್‌ಲೈನ್‌ನಲ್ಲಿ ಬಳಸಲು ಆಯ್ಕೆ ಮಾಡಬಹುದು - ಉದಾಹರಣೆಗೆ, ಪಠ್ಯ ಉತ್ಪಾದನೆ. [ಮಾಡೆಲ್ ಹಬ್‌ಗೆ](https://huggingface.co/models) ಹೋಗಿ ಮತ್ತು ಆ ಕಾರ್ಯಕ್ಕೆ ಬೆಂಬಲಿತ ಮಾದರಿಗಳನ್ನು ಮಾತ್ರ ಪ್ರದರ್ಶಿಸಲು ಎಡಭಾಗದಲ್ಲಿರುವ ಅನುಗುಣವಾದ ಟ್ಯಾಗ್ ಅನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ. ನೀವು [ಈ ರೀತಿಯ](https://huggingface.co/models?pipeline_tag=text-generation) ಪುಟವನ್ನು ಪಡೆಯಬೇಕು.

[`HuggingFaceTB/SmolLM2-360M`](https://huggingface.co/HuggingFaceTB/SmolLM2-360M) ಮಾದರಿಯನ್ನು ಪ್ರಯತ್ನಿಸೋಣ! ಮೊದಲಿನಂತೆಯೇ ಅದೇ ಪೈಪ್‌ಲೈನ್‌ನಲ್ಲಿ ಅದನ್ನು ಲೋಡ್ ಮಾಡುವುದು ಹೇಗೆ ಎಂಬುದು ಇಲ್ಲಿದೆ:

```python
from transformers import pipeline

generator = pipeline("text-generation", model="HuggingFaceTB/SmolLM2-360M")
generator(
    "In this course, we will teach you how to",
    max_length=30,
    num_return_sequences=2,
)
```

```python out
[{'generated_text': 'In this course, we will teach you how to manipulate the world and '
                    'move your mental and physical capabilities to your advantage.'},
 {'generated_text': 'In this course, we will teach you how to become an expert and '
                    'practice realtime, and with a hands on experience on both real '
                    'time and real'}]
```

ಭಾಷಾ ಟ್ಯಾಗ್‌ಗಳ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡುವ ಮೂಲಕ ನೀವು ಮಾದರಿಗಾಗಿ ನಿಮ್ಮ ಹುಡುಕಾಟವನ್ನು ಪರಿಷ್ಕರಿಸಬಹುದು ಮತ್ತು ಇನ್ನೊಂದು ಭಾಷೆಯಲ್ಲಿ ಪಠ್ಯವನ್ನು ರಚಿಸುವ ಮಾದರಿಯನ್ನು ಆಯ್ಕೆ ಮಾಡಬಹುದು. ಮಾಡೆಲ್ ಹಬ್ ಹಲವಾರು ಭಾಷೆಗಳನ್ನು ಬೆಂಬಲಿಸುವ ಬಹುಭಾಷಾ ಮಾದರಿಗಳಿಗಾಗಿ ಚೆಕ್‌ಪಾಯಿಂಟ್‌ಗಳನ್ನು ಸಹ ಒಳಗೊಂಡಿದೆ.

ನೀವು ಅದರ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡುವ ಮೂಲಕ ಮಾದರಿಯನ್ನು ಆಯ್ಕೆ ಮಾಡಿದ ನಂತರ, ಅದನ್ನು ನೇರವಾಗಿ ಆನ್‌ಲೈನ್‌ನಲ್ಲಿ ಪ್ರಯತ್ನಿಸಲು ನಿಮಗೆ ಅನುವು ಮಾಡಿಕೊಡುವ ವಿಜೆಟ್ ಇದೆ ಎಂದು ನೀವು ನೋಡುತ್ತೀರಿ. ಈ ರೀತಿಯಾಗಿ ನೀವು ಅದನ್ನು ಡೌನ್‌ಲೋಡ್ ಮಾಡುವ ಮೊದಲು ಮಾದರಿಯ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ತ್ವರಿತವಾಗಿ ಪರೀಕ್ಷಿಸಬಹುದು.

> [!TIP]
> ✏️ **ಇದನ್ನು ಪ್ರಯತ್ನಿಸಿ!** ಇನ್ನೊಂದು ಭಾಷೆಗೆ ಪಠ್ಯ ಉತ್ಪಾದನೆ ಮಾದರಿಯನ್ನು ಕಂಡುಹಿಡಿಯಲು ಫಿಲ್ಟರ್‌ಗಳನ್ನು ಬಳಸಿ. ವಿಜೆಟ್‌ನೊಂದಿಗೆ ಆಡಲು ಮತ್ತು ಅದನ್ನು ಪೈಪ್‌ಲೈನ್‌ನಲ್ಲಿ ಬಳಸಲು ಹಿಂಜರಿಯಬೇಡಿ!

### ತೀರ್ಮಾನ ಪೂರೈಕೆದಾರರು[[ತೀರ್ಮಾನ ಪೂರೈಕೆದಾರರು]]

[ಹಗ್ಗಿಂಗ್ ಫೇಸ್ ವೆಬ್‌ಸೈಟ್‌ನಲ್ಲಿ](https://huggingface.co/docs/inference-providers/en/index) ಲಭ್ಯವಿರುವ ಇನ್ಫರೆನ್ಸ್ ಪ್ರೊವೈಡರ್‌ಗಳನ್ನು ಬಳಸಿಕೊಂಡು ಎಲ್ಲಾ ಮಾದರಿಗಳನ್ನು ನಿಮ್ಮ ಬ್ರೌಸರ್ ಮೂಲಕ ನೇರವಾಗಿ ಪರೀಕ್ಷಿಸಬಹುದು. ಕಸ್ಟಮ್ ಪಠ್ಯವನ್ನು ಇನ್‌ಪುಟ್ ಮಾಡುವ ಮೂಲಕ ಮತ್ತು ಮಾದರಿಯು ಇನ್‌ಪುಟ್ ಡೇಟಾವನ್ನು ಪ್ರಕ್ರಿಯೆಗೊಳಿಸುವುದನ್ನು ವೀಕ್ಷಿಸುವ ಮೂಲಕ ನೀವು ಈ ಪುಟದಲ್ಲಿ ನೇರವಾಗಿ ಮಾದರಿಯೊಂದಿಗೆ ಆಟವಾಡಬಹುದು.

ವಿಜೆಟ್‌ಗೆ ಶಕ್ತಿ ನೀಡುವ ಇನ್ಫರೆನ್ಸ್ ಪ್ರೊವೈಡರ್‌ಗಳು ಪಾವತಿಸಿದ ಉತ್ಪನ್ನವಾಗಿಯೂ ಲಭ್ಯವಿದೆ, ಇದು ನಿಮ್ಮ ಕೆಲಸದ ಹರಿವುಗಳಿಗೆ ಅಗತ್ಯವಿದ್ದರೆ ಸೂಕ್ತವಾಗಿ ಬರುತ್ತದೆ. ಹೆಚ್ಚಿನ ವಿವರಗಳಿಗಾಗಿ [ಬೆಲೆ ಪುಟವನ್ನು](https://huggingface.co/docs/inference-providers/en/pricing) ನೋಡಿ.

## ಮುಖವಾಡ ತುಂಬುವುದು[[ಮುಖವಾಡ ತುಂಬುವುದು]]

ನೀವು ಪ್ರಯತ್ನಿಸುವ ಮುಂದಿನ ಪೈಪ್‌ಲೈನ್ `ಫಿಲ್-ಮಾಸ್ಕ್` ಆಗಿದೆ. ಈ ಕಾರ್ಯದ ಉದ್ದೇಶವು ನಿರ್ದಿಷ್ಟ ಪಠ್ಯದಲ್ಲಿನ ಖಾಲಿ ಜಾಗಗಳನ್ನು ಭರ್ತಿ ಮಾಡುವುದು:

```python
from transformers import pipeline

unmasker = pipeline("fill-mask")
unmasker("This course will teach you all about <mask> models.", top_k=2)
```

```python out
[{'sequence': 'This course will teach you all about mathematical models.',
  'score': 0.19619831442832947,
  'token': 30412,
  'token_str': ' mathematical'},
 {'sequence': 'This course will teach you all about computational models.',
  'score': 0.04052725434303284,
  'token': 38163,
  'token_str': ' computational'}]
```

`top_k` ಆರ್ಗ್ಯುಮೆಂಟ್ ನಿಮಗೆ ಎಷ್ಟು ಸಾಧ್ಯತೆಗಳನ್ನು ಪ್ರದರ್ಶಿಸಲು ಬಯಸುತ್ತದೆ ಎಂಬುದನ್ನು ನಿಯಂತ್ರಿಸುತ್ತದೆ. ಇಲ್ಲಿ ಮಾದರಿಯು ವಿಶೇಷ `<mask>` ಪದವನ್ನು ತುಂಬುತ್ತದೆ ಎಂಬುದನ್ನು ಗಮನಿಸಿ, ಇದನ್ನು ಹೆಚ್ಚಾಗಿ ಮಾಸ್ಕ್ ಟೋಕನ್ ಎಂದು ಕರೆಯಲಾಗುತ್ತದೆ. ಇತರ ಮಾಸ್ಕ್-ಫಿಲ್ಲಿಂಗ್ ಮಾದರಿಗಳು ವಿಭಿನ್ನ ಮಾಸ್ಕ್ ಟೋಕನ್‌ಗಳನ್ನು ಹೊಂದಿರಬಹುದು, ಆದ್ದರಿಂದ ಇತರ ಮಾದರಿಗಳನ್ನು ಅನ್ವೇಷಿಸುವಾಗ ಸರಿಯಾದ ಮಾಸ್ಕ್ ಪದವನ್ನು ಪರಿಶೀಲಿಸುವುದು ಯಾವಾಗಲೂ ಒಳ್ಳೆಯದು. ಅದನ್ನು ಪರಿಶೀಲಿಸಲು ಒಂದು ಮಾರ್ಗವೆಂದರೆ ವಿಜೆಟ್‌ನಲ್ಲಿ ಬಳಸಲಾದ ಮಾಸ್ಕ್ ಪದವನ್ನು ನೋಡುವುದು.

> [!TIP]
> ✏️ **ಇದನ್ನು ಪ್ರಯತ್ನಿಸಿ! **ಹಬ್‌ನಲ್ಲಿ `bert-base-cased` ಮಾದರಿಯನ್ನು ಹುಡುಕಿ ಮತ್ತು ಇನ್ಫರೆನ್ಸ್ API ವಿಜೆಟ್‌ನಲ್ಲಿ ಅದರ ಮಾಸ್ಕ್ ಪದವನ್ನು ಗುರುತಿಸಿ. ಮೇಲಿನ ನಮ್ಮ `ಪೈಪ್‌ಲೈನ್` ಉದಾಹರಣೆಯಲ್ಲಿರುವ ವಾಕ್ಯಕ್ಕೆ ಈ ಮಾದರಿಯು ಏನನ್ನು ಊಹಿಸುತ್ತದೆ?

## Named entity recognition[[named-entity-recognition]]

ನೇಮ್ಡ್ ಎಂಟಿಟಿ ರೆಕಗ್ನಿಷನ್ (NER) ಎನ್ನುವುದು ಮಾದರಿಯು ಇನ್‌ಪುಟ್ ಪಠ್ಯದ ಯಾವ ಭಾಗಗಳು ವ್ಯಕ್ತಿಗಳು, ಸ್ಥಳಗಳು ಅಥವಾ ಸಂಸ್ಥೆಗಳಂತಹ ಎಂಟಿಟಿಗಳಿಗೆ ಹೊಂದಿಕೆಯಾಗುತ್ತವೆ ಎಂಬುದನ್ನು ಕಂಡುಹಿಡಿಯಬೇಕಾದ ಕಾರ್ಯವಾಗಿದೆ. ಒಂದು ಉದಾಹರಣೆಯನ್ನು ನೋಡೋಣ:

```python
from transformers import pipeline

ner = pipeline("ner", aggregation_strategy="simple")
ner("My name is Sylvain and I work at Hugging Face in Brooklyn.")
```

```python out
[{'entity_group': 'PER', 'score': 0.99816, 'word': 'Sylvain', 'start': 11, 'end': 18}, 
 {'entity_group': 'ORG', 'score': 0.97960, 'word': 'Hugging Face', 'start': 33, 'end': 45}, 
 {'entity_group': 'LOC', 'score': 0.99321, 'word': 'Brooklyn', 'start': 49, 'end': 57}
]
```

ಇಲ್ಲಿ ಮಾದರಿಯು ಸಿಲ್ವೈನ್ ಒಬ್ಬ ವ್ಯಕ್ತಿ (PER), ಹಗ್ಗಿಂಗ್ ಫೇಸ್ ಒಂದು ಸಂಸ್ಥೆ (ORG), ಮತ್ತು ಬ್ರೂಕ್ಲಿನ್ ಒಂದು ಸ್ಥಳ (LOC) ಎಂದು ಸರಿಯಾಗಿ ಗುರುತಿಸಿದೆ.

ಪೈಪ್‌ಲೈನ್ ಸೃಷ್ಟಿ ಕಾರ್ಯದಲ್ಲಿ ನಾವು `aggregation_strategy="simple"` ಆಯ್ಕೆಯನ್ನು ರವಾನಿಸುತ್ತೇವೆ, ಇದು ಪೈಪ್‌ಲೈನ್‌ಗೆ ಒಂದೇ ಘಟಕಕ್ಕೆ ಅನುಗುಣವಾದ ವಾಕ್ಯದ ಭಾಗಗಳನ್ನು ಮರುಸಂಗ್ರಹಿಸಲು ಹೇಳುತ್ತದೆ: ಇಲ್ಲಿ ಮಾದರಿಯು "ಹಗ್ಗಿಂಗ್" ಮತ್ತು "ಫೇಸ್" ಅನ್ನು ಒಂದೇ ಸಂಘಟನೆಯಾಗಿ ಸರಿಯಾಗಿ ಗುಂಪು ಮಾಡಿದೆ, ಆದರೂ ಹೆಸರು ಬಹು ಪದಗಳನ್ನು ಒಳಗೊಂಡಿದೆ. ವಾಸ್ತವವಾಗಿ, ಮುಂದಿನ ಅಧ್ಯಾಯದಲ್ಲಿ ನಾವು ನೋಡಲಿರುವಂತೆ, ಪೂರ್ವ-ಸಂಸ್ಕರಣೆಯು ಕೆಲವು ಪದಗಳನ್ನು ಸಣ್ಣ ಭಾಗಗಳಾಗಿ ವಿಭಜಿಸುತ್ತದೆ. ಉದಾಹರಣೆಗೆ, `ಸಿಲ್ವೈನ್` ಅನ್ನು ನಾಲ್ಕು ತುಣುಕುಗಳಾಗಿ ವಿಂಗಡಿಸಲಾಗಿದೆ: `S`, `##yl`, `##va`, ಮತ್ತು `##in`. ನಂತರದ ಸಂಸ್ಕರಣಾ ಹಂತದಲ್ಲಿ, ಪೈಪ್‌ಲೈನ್ ಆ ತುಣುಕುಗಳನ್ನು ಯಶಸ್ವಿಯಾಗಿ ಮರುಸಂಘಟಿಸಿತು.

> [!TIP]
> ✏️ **ಇದನ್ನು ಪ್ರಯತ್ನಿಸಿ!** ಇಂಗ್ಲಿಷ್‌ನಲ್ಲಿ ಭಾಷಣದ ಭಾಗ ಟ್ಯಾಗಿಂಗ್ (ಸಾಮಾನ್ಯವಾಗಿ POS ಎಂದು ಸಂಕ್ಷಿಪ್ತಗೊಳಿಸಲಾಗುತ್ತದೆ) ಮಾಡಲು ಸಾಧ್ಯವಾಗುವ ಮಾದರಿಗಾಗಿ ಮಾದರಿ ಹಬ್ ಅನ್ನು ಹುಡುಕಿ. ಮೇಲಿನ ಉದಾಹರಣೆಯಲ್ಲಿ ವಾಕ್ಯಕ್ಕಾಗಿ ಈ ಮಾದರಿ ಏನು ಊಹಿಸುತ್ತದೆ?

## ಪ್ರಶ್ನೋತ್ತರ [[ಪ್ರಶ್ನೋತ್ತರ ]]

`question-answering` ಪೈಪ್‌ಲೈನ್ ನಿರ್ದಿಷ್ಟ ಸಂದರ್ಭದಿಂದ ಮಾಹಿತಿಯನ್ನು ಬಳಸಿಕೊಂಡು ಪ್ರಶ್ನೆಗಳಿಗೆ ಉತ್ತರಿಸುತ್ತದೆ:

```python
from transformers import pipeline

question_answerer = pipeline("question-answering")
question_answerer(
    question="Where do I work?",
    context="My name is Sylvain and I work at Hugging Face in Brooklyn",
)
```

```python out
{'score': 0.6385916471481323, 'start': 33, 'end': 45, 'answer': 'Hugging Face'}
```

ಒದಗಿಸಲಾದ ಸಂದರ್ಭದಿಂದ ಮಾಹಿತಿಯನ್ನು ಹೊರತೆಗೆಯುವ ಮೂಲಕ ಈ ಪೈಪ್‌ಲೈನ್ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ ಎಂಬುದನ್ನು ಗಮನಿಸಿ; ಇದು ಉತ್ತರವನ್ನು ಉತ್ಪಾದಿಸುವುದಿಲ್ಲ.

## ಸಾರಾಂಶ[[ಸಾರಾಂಶ]]

ಸಾರಾಂಶೀಕರಣವು ಪಠ್ಯದಲ್ಲಿ ಉಲ್ಲೇಖಿಸಲಾದ ಎಲ್ಲಾ (ಅಥವಾ ಹೆಚ್ಚಿನ) ಪ್ರಮುಖ ಅಂಶಗಳನ್ನು ಉಳಿಸಿಕೊಂಡು ಪಠ್ಯವನ್ನು ಚಿಕ್ಕ ಪಠ್ಯವಾಗಿ ಕಡಿಮೆ ಮಾಡುವ ಕಾರ್ಯವಾಗಿದೆ. ಒಂದು ಉದಾಹರಣೆ ಇಲ್ಲಿದೆ:

```python
from transformers import pipeline

summarizer = pipeline("summarization")
summarizer(
    """
    America has changed dramatically during recent years. Not only has the number of 
    graduates in traditional engineering disciplines such as mechanical, civil, 
    electrical, chemical, and aeronautical engineering declined, but in most of 
    the premier American universities engineering curricula now concentrate on 
    and encourage largely the study of engineering science. As a result, there 
    are declining offerings in engineering subjects dealing with infrastructure, 
    the environment, and related issues, and greater concentration on high 
    technology subjects, largely supporting increasingly complex scientific 
    developments. While the latter is important, it should not be at the expense 
    of more traditional engineering.

    Rapidly developing economies such as China and India, as well as other 
    industrial countries in Europe and Asia, continue to encourage and advance 
    the teaching of engineering. Both China and India, respectively, graduate 
    six and eight times as many traditional engineers as does the United States. 
    Other industrial countries at minimum maintain their output, while America 
    suffers an increasingly serious decline in the number of engineering graduates 
    and a lack of well-educated engineers.
"""
)
```

```python out
[{'summary_text': ' America has changed dramatically during recent years . The '
                  'number of engineering graduates in the U.S. has declined in '
                  'traditional engineering disciplines such as mechanical, civil '
                  ', electrical, chemical, and aeronautical engineering . Rapidly '
                  'developing economies such as China and India, as well as other '
                  'industrial countries in Europe and Asia, continue to encourage '
                  'and advance engineering .'}]
```

ಪಠ್ಯ ರಚನೆಯಂತೆ, ನೀವು ಫಲಿತಾಂಶಕ್ಕಾಗಿ `max_length` ಅಥವಾ `min_length` ಅನ್ನು ನಿರ್ದಿಷ್ಟಪಡಿಸಬಹುದು.

## ಅನುವಾದ[[ಅನುವಾದ]]

ಅನುವಾದಕ್ಕಾಗಿ, ನೀವು ಕಾರ್ಯ ಹೆಸರಿನಲ್ಲಿ ಭಾಷಾ ಜೋಡಿಯನ್ನು ಒದಗಿಸಿದರೆ (`"translation_en_to_fr"` ನಂತಹ) ನೀವು ಡೀಫಾಲ್ಟ್ ಮಾದರಿಯನ್ನು ಬಳಸಬಹುದು, ಆದರೆ [ಮಾದರಿ ಹಬ್‌ನಲ್ಲಿ](https://huggingface.co/models) ನೀವು ಬಳಸಲು ಬಯಸುವ ಮಾದರಿಯನ್ನು ಆರಿಸುವುದು ಸುಲಭವಾದ ಮಾರ್ಗವಾಗಿದೆ. ಇಲ್ಲಿ ನಾವು ಫ್ರೆಂಚ್‌ನಿಂದ ಇಂಗ್ಲಿಷ್‌ಗೆ ಅನುವಾದಿಸಲು ಪ್ರಯತ್ನಿಸುತ್ತೇವೆ:

```python
from transformers import pipeline

translator = pipeline("translation", model="Helsinki-NLP/opus-mt-fr-en")
translator("Ce cours est produit par Hugging Face.")
```

```python out
[{'translation_text': 'This course is produced by Hugging Face.'}]
```

ಪಠ್ಯ ರಚನೆ ಮತ್ತು ಸಾರಾಂಶದಂತೆ, ನೀವು ಫಲಿತಾಂಶಕ್ಕಾಗಿ `max_length` ಅಥವಾ `min_length` ನಿರ್ದಿಷ್ಟಪಡಿಸಬಹುದು.

> [!TIP]
> ✏️ **ಇದನ್ನು ಪ್ರಯತ್ನಿಸಿ!** ಇತರ ಭಾಷೆಗಳಲ್ಲಿ ಅನುವಾದ ಮಾದರಿಗಳನ್ನು ಹುಡುಕಿ ಮತ್ತು ಹಿಂದಿನ ವಾಕ್ಯವನ್ನು ಕೆಲವು ವಿಭಿನ್ನ ಭಾಷೆಗಳಿಗೆ ಅನುವಾದಿಸಲು ಪ್ರಯತ್ನಿಸಿ.

## ಬಹು ಮೂಲಗಳಿಂದ ಡೇಟಾವನ್ನು ಸಂಯೋಜಿಸುವುದು

ಟ್ರಾನ್ಸ್‌ಫಾರ್ಮರ್ ಮಾದರಿಗಳ ಒಂದು ಪ್ರಬಲ ಅನ್ವಯವೆಂದರೆ ಬಹು ಮೂಲಗಳಿಂದ ಡೇಟಾವನ್ನು ಸಂಯೋಜಿಸುವ ಮತ್ತು ಪ್ರಕ್ರಿಯೆಗೊಳಿಸುವ ಅವುಗಳ ಸಾಮರ್ಥ್ಯ. ನಿಮಗೆ ಅಗತ್ಯವಿರುವಾಗ ಇದು ವಿಶೇಷವಾಗಿ ಉಪಯುಕ್ತವಾಗಿದೆ:

1. ಬಹು ಡೇಟಾಬೇಸ್‌ಗಳು ಅಥವಾ ರೆಪೊಸಿಟರಿಗಳಲ್ಲಿ ಹುಡುಕಿ
2. ವಿವಿಧ ಸ್ವರೂಪಗಳಿಂದ (ಪಠ್ಯ, ಚಿತ್ರಗಳು, ಆಡಿಯೋ) ಮಾಹಿತಿಯನ್ನು ಕ್ರೋಢೀಕರಿಸಿ
3. ಸಂಬಂಧಿತ ಮಾಹಿತಿಯ ಏಕೀಕೃತ ನೋಟವನ್ನು ರಚಿಸಿ

ಉದಾಹರಣೆಗೆ, ನೀವು ಈ ಕೆಳಗಿನ ವ್ಯವಸ್ಥೆಯನ್ನು ನಿರ್ಮಿಸಬಹುದು:

- ಪಠ್ಯ ಮತ್ತು ಚಿತ್ರದಂತಹ ಬಹು ವಿಧಾನಗಳಲ್ಲಿ ಡೇಟಾಬೇಸ್‌ಗಳಲ್ಲಿ ಮಾಹಿತಿಯನ್ನು ಹುಡುಕುತ್ತದೆ.
- ವಿಭಿನ್ನ ಮೂಲಗಳಿಂದ ಫಲಿತಾಂಶಗಳನ್ನು ಒಂದೇ ಸುಸಂಬದ್ಧ ಪ್ರತಿಕ್ರಿಯೆಯಾಗಿ ಸಂಯೋಜಿಸುತ್ತದೆ. ಉದಾಹರಣೆಗೆ, ಆಡಿಯೋ ಫೈಲ್ ಮತ್ತು ಪಠ್ಯ ವಿವರಣೆಯಿಂದ.
- ದಾಖಲೆಗಳು ಮತ್ತು ಮೆಟಾಡೇಟಾದ ಡೇಟಾಬೇಸ್‌ನಿಂದ ಹೆಚ್ಚು ಸೂಕ್ತವಾದ ಮಾಹಿತಿಯನ್ನು ಪ್ರಸ್ತುತಪಡಿಸುತ್ತದೆ.

## ಸಮಾಪ್ತಿ

ಈ ಅಧ್ಯಾಯದಲ್ಲಿ ತೋರಿಸಿರುವ ಪೈಪ್‌ಲೈನ್‌ಗಳು ಹೆಚ್ಚಾಗಿ ಪ್ರದರ್ಶನ ಉದ್ದೇಶಗಳಿಗಾಗಿವೆ. ಅವುಗಳನ್ನು ನಿರ್ದಿಷ್ಟ ಕಾರ್ಯಗಳಿಗಾಗಿ ಪ್ರೋಗ್ರಾಮ್ ಮಾಡಲಾಗಿದೆ ಮತ್ತು ಅವುಗಳ ವ್ಯತ್ಯಾಸಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಸಾಧ್ಯವಿಲ್ಲ. ಮುಂದಿನ ಅಧ್ಯಾಯದಲ್ಲಿ, ಪೈಪ್‌ಲೈನ್() ಕಾರ್ಯದೊಳಗೆ ಏನಿದೆ ಮತ್ತು ಅದರ ನಡವಳಿಕೆಯನ್ನು ಹೇಗೆ ಕಸ್ಟಮೈಸ್ ಮಾಡುವುದು ಎಂಬುದನ್ನು ನೀವು ಕಲಿಯುವಿರಿ.
